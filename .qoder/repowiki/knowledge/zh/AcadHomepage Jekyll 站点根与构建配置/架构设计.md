本仓库为单站点的 Jekyll 工程根，无子模块；所有站点源码位于同级目录（如 `_pages`、`_sass`、`assets`），由根级配置文件驱动：
- `_config.yml` 定义站点元信息（title、author、repository）、Markdown 处理器（kramdown + GFM）、Sass 路径（`_sass`、`assets/css`）、permalink 格式、时区（Asia/Shanghai）以及 jekyll-paginate / jekyll-sitemap / jekyll-gist / jekyll-feed / jekyll-redirect-from 等插件白名单。
- `Gemfile` 锁定 Jekyll ~>4.3 及 kramdown/rouge/minima/tzinfo 等运行时依赖，并通过 `group :jekyll_plugins` 声明插件 gem，兼容 Ruby 3.4 移除的默认 gem（csv、base64、logger、mutex_m、bigdecimal、drb、timeout、webrick）。
- `vercel.json` 通过 SPA rewrites 将所有路由重写到 `index.html`，使 Jekyll 生成的静态页在 Vercel 上以单页应用方式运行。
- `run_server.sh` 封装 `bundle exec jekyll serve --livereload`，作为本地开发入口。
- `.gitignore` / `.vercelignore` 分别屏蔽本地构建产物与 Vercel 构建缓存，避免污染发布包。
依赖方向单向：脚本 → bundle → Jekyll → 主题（minima/AcadHomepage）→ 站点内容，外部仅通过 GitHub Pages/Vercel 触发构建。
